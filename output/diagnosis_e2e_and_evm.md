## 3. 予実管理による粗利向上の検討

### 3.1 粗利向上のメカニズム（理論）

```
【従来の管理】
  月末に手作業で確認 → 超過発見が遅い → 是正間に合わず → 粗利喪失

         |   予実管理導入   |

【予実管理導入後】
  入力のたびに消化率自動計算 → 即時信号表示 → 早期是正 → 粗利防衛
```

予実管理が粗利に効く理由は「**予算超過の早期検知による損失回避**」にある。
粗利を「増やす」のではなく、「失わない」仕組み。

予実管理の主要指標:
- **消化率** = 累計支払額 / 実行予算額。100%超は予算超過
- **出来高率** = 出来高（進捗） / 実行予算額。工事の完成度合い
- **過不足見込み** = 実行予算額 - (累計支払額 / 出来高率 x 100%)。正値=余る、負値=足りない
- **信号判定**: 消化率と出来高率の乖離に基づく3段階（青/黄/赤）

粗利率との関係:
- **予測粗利率** = 目標粗利率 - (消化率 - 出来高率) x 調整係数
- 消化率 > 出来高率 の場合、予算を工程より速く消費しており、粗利の喪失に直結

#### 粗利シミュレーション

```
工事価格（税抜）:       39,840,000円
目標粗利率:            15.0%
目標粗利額:            5,976,000円    = 39,840,000 x 15.0%

--- 予実管理未導入（現状推定） ---

超過費目の合計過不足:   -1,818,689円
  内訳:
    護岸工外注費       -852,174円  （消化率75.8%で既に黄信号）
    排水工外注費       -757,500円  （消化率93.8%、赤信号）
    法定福利費         -182,163円  （消化率126.2%、最も深刻）
    地代家賃            -17,988円  （消化率113.7%）
    水道光熱費           -8,864円  （消化率113.0%）

  → 実現粗利: 5,976,000 - 1,818,689 = 4,157,311円
  → 実現粗利率: 10.4%
  → 粗利率低下: -4.6pt

--- 予実管理導入後（早期検知で超過を50%抑制と仮定） ---

抑制対象: 超過費目の合計 -1,818,689円
抑制効果: 1,818,689 x 50% = 909,345円
  → 粗利回復: +909,345円
  → 実現粗利: 4,157,311 + 909,345 = 5,066,656円
  → 実現粗利率: 12.7% (+2.3pt改善)

--- 比較まとめ ---

| シナリオ | 粗利額 | 粗利率 |
|---------|-------:|-------:|
| 目標（予算基準） | 5,976,000円 | 15.0% |
| 予実管理未導入（超過放置） | 4,157,311円 | 10.4% |
| 予実管理導入（50%抑制） | 5,066,656円 | 12.7% |
| **予実管理導入の改善効果** | **+909,345円** | **+2.3pt** |
```

**補足**: 50%抑制の根拠は「消化率100%超の警告を月初に受け取れば、月中に手を打てる」という前提。法定福利費（消化率126.2%）のように構造的に超過している費目は抑制困難だが、護岸工外注費（消化率75.8%で黄信号）のような早期警告段階なら業者との調整で改善余地がある。

### 3.3 粗利向上の条件（予実管理が効くために必要なこと）

予実管理は「仕組み」だけでは粗利に効かない。以下の **4つの運用条件** が必須:

| 条件 | 内容 | 現行の準備状況 |
|------|------|--------------|
| 1. 毎日見る | 所長がダッシュボードを日次で確認し、赤信号の費目に気づく | ダッシュボード設計済み。Google Sites公開で即アクセス可能 |
| 2. 即座に対策 | 信号が黄→赤に変わったら対策を打つ。「排水工が赤信号」→残工事の外注交渉 | 予算超過アラートのメール通知を実装済み（config.gs） |
| 3. 月次レビュー | 月次で過不足見込みを振り返り、翌月の予算調整を実行 | 月次スナップショット方式で推移記録（aggregation.gs） |
| 4. 横串比較 | 経営層が工事横断で消化率を比較し、効率的な現場のノウハウを展開 | hub.gs骨格あり。予算健康度連携は未対応 |

### 3.4 結論: 粗利は向上するか？

**条件付きで向上する。**

- 予実管理の本質は「損失回避」。粗利を積極的に増やすツールではなく、**予算超過による粗利喪失を防ぐ**ツール
- 海潟漁港R7-1の試算では、予実管理導入により **+909,345円（+2.3pt）の粗利改善** が見込まれる
- 建設業の粗利率は通常10-20%。超過検知が1ヶ月早まるだけで、工事あたり数十万から数百万円の改善余地がある
- 全6工事（合計請負額4.2億円）に展開した場合、年間で **数百万円規模の粗利防衛効果** が期待できる

ただし「システムがあるだけ」では効かない。**運用（毎日見る、即座に対策する、月次で振り返る）が必須**。
